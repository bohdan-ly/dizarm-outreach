# Deploying outreach-ai to Vercel

## Prerequisites

| Requirement | Why |
|-------------|-----|
| Vercel account (Pro plan) | Minute-level cron requires Pro. Hobby only allows once-daily cron. |
| Vercel CLI installed | `npm i -g vercel` |
| GitHub repo for this project | Vercel deploys from Git. Push `outreach-ai/` as a standalone repo. |
| Notion integration token | Already in `.env` — just needs to be added to Vercel's env vars. |
| Anthropic API key | Same. |

---

## Step 1 — Push this project as a standalone GitHub repo

Vercel deploys one Git repo per project. The `outreach-ai/` folder needs its own repo.

```bash
cd projects/outreach-ai
git init
git add .
git commit -m "init: prepare for Vercel deployment"
gh repo create outreach-ai --private --source=. --push
```

---

## Step 2 — Install @vercel/kv dependency

`learnings.json` cannot persist on serverless. The app uses Vercel KV (Redis) instead.

```bash
npm install
```

The `@vercel/kv` package is already in `package.json`. This installs it.

---

## Step 3 — Create a Vercel project

```bash
vercel
```

When prompted:
- Link to existing project? **No**
- Project name: `outreach-ai`
- Root directory: `.` (current directory — `outreach-ai/`)
- Override build settings? **No**

Vercel will auto-detect the `/api` directory and deploy each file as a serverless function.

---

## Step 4 — Create a Vercel KV store

The learning cycle persists reply patterns in Vercel KV (Redis). Do this once.

1. Go to [vercel.com/dashboard](https://vercel.com/dashboard)
2. Open your `outreach-ai` project
3. Click **Storage** tab
4. Click **Create Database** > select **KV**
5. Name it `outreach-ai-kv`
6. Click **Connect to Project** and select `outreach-ai`

Vercel automatically injects these env vars into all environments:
- `KV_URL`
- `KV_REST_API_URL`
- `KV_REST_API_TOKEN`
- `KV_REST_API_READ_ONLY_TOKEN`

No manual copy-paste needed.

---

## Step 5 — Set environment variables in Vercel

Go to your project **Settings > Environment Variables** and add:

| Variable | Value | Environments |
|----------|-------|--------------|
| `NOTION_TOKEN` | your Notion integration token | Production, Preview, Development |
| `ANTHROPIC_API_KEY` | your Anthropic API key | Production, Preview, Development |

`CRON_SECRET` is auto-generated by Vercel — do NOT set it manually.
KV variables are already injected from Step 4.

---

## Step 6 — Deploy to production

```bash
vercel --prod
```

Or push to your main branch and Vercel will auto-deploy via the GitHub integration.

After deploy, verify endpoints are alive:

```bash
curl https://your-project.vercel.app/api/health
# Expected: {"status":"ok","service":"outreach-ai"}
```

---

## Step 7 — Verify cron jobs are registered

1. Go to Vercel dashboard > your project > **Cron Jobs** tab
2. You should see two cron jobs:

| Path | Schedule | What it does |
|------|----------|--------------|
| `/api/cron/scan` | `* * * * *` (every minute) | Scans Notion for 🤖🤖🤖 leads and generates cover letters |
| `/api/cron/learn` | `0 * * * *` (every hour) | Analyzes replied messages and updates writing patterns |

**Important:** Minute-level cron (`* * * * *`) requires Vercel Pro. On Hobby plan, the minimum is once daily. If you're on Hobby, change the scan schedule in `vercel.json` to `0 0 * * *` and use the webhook endpoint for on-demand triggering.

---

## Step 8 — Test the webhook manually

Trigger a scan immediately (without waiting for the cron):

```bash
# Scan all 🤖🤖🤖 leads
curl -X POST https://your-project.vercel.app/api/webhook \
  -H "Content-Type: application/json" \
  -d '{}'

# Process a single lead by Notion page ID
curl -X POST https://your-project.vercel.app/api/webhook \
  -H "Content-Type: application/json" \
  -d '{"pageId":"your-notion-page-id"}'

# Trigger a learning cycle manually
curl -X POST https://your-project.vercel.app/api/learn \
  -H "Content-Type: application/json" \
  -d '{}'
```

## API reference

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/health` | GET | Health check |
| `/api/webhook` | POST | Scan all 🤖 leads, or process `{"pageId":"..."}` |
| `/api/learn` | POST | Run learning cycle manually |
| `/api/cron/scan` | GET | Called by Vercel cron every minute |
| `/api/cron/learn` | GET | Called by Vercel cron every hour |

## Local development (unchanged)

The original `index.js` still works for local dev. It uses `node-cron` and reads/writes `learnings.json`. Nothing changes locally.

```bash
cp .env.example .env
# fill in NOTION_TOKEN and ANTHROPIC_API_KEY
npm run dev
```

On Vercel, `KV_REST_API_URL` is set so learnings go to KV. Locally it's not set, so the local file is used. No config needed to switch between modes.

## Troubleshooting

**Cron jobs not firing:**
- Check Vercel project > Cron Jobs tab for last run and errors
- Confirm you are on Pro plan for minute-level cron
- Check function logs under Deployments > Functions

**KV not persisting:**
- Confirm KV database is connected to the project in Storage tab
- Check that `KV_REST_API_URL` appears in Settings > Environment Variables (auto-injected)

**Notion errors in logs:**
- Confirm `NOTION_TOKEN` is set in Vercel env vars
- Confirm the integration has access to both the Leads database and Messages database in Notion

**`@vercel/kv` import error locally:**
- This is expected. Locally, `KV_REST_API_URL` is not set so KV is never imported. The local file fallback runs instead.
